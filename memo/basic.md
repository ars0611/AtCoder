# 実装時に気を付ける基礎的なこと
- .join()は``str型``、もしくは``str型を持つ配列``にしか使えない   
- rangeは [start, stop) である   
- ある数の二乗根を超えない最大の整数が欲しければ、``int(math.sqrt(num))``は**誤差がでる**ため、``math.isqrt(num)``を用いましょう   
- 除算でオーバーフローの可能性があるなら、式変形をして整数で扱えるようにする   
```python: example.py
if a / b == c / d:  # 正しい結果が得られない可能性がある
if a * d == c * b:   # 上記と同じ条件式だが、常に正しい結果が得られる
``` 
- 桁数がとても大きい数の演算は計算に時間がかかるため、modで解答する場合は、演算ごとに**毎回modを取ろう**
- pythonはintが多倍長整数なので、**大きすぎる値を大量に抱えると**MLEになる。そういった面でも、modはマメにとってあげるべき   
- 計算量の見積もりは雰囲気ではなく丁寧にやろう
- クエリを逆順に取得したら,出力も逆順にしましょう
- 配列が持つデータの型は**一意**に定めるべきで、boolとintを持つ配列などはとても重たい   
- いくつかのケースが落とされているときは、嘘解法であると棄却する前に落ちるケースを手元で考えよう   
- while num < math.sqrt(tmp) などするとループのたびにモジュールを呼び出してTLEするのでwhileの前で変数でもっておこう   
```python: example.py
# 遅い実装
num = 10**12
i = 0
while i < math.isqrt(num): #ループを続けるかの判定の際に毎回math.isqrt(num)を計算している
```
```python: example.py
# 速い実装
num = 10**12
root_num = math.isqrt(num) # この一回のみ
while i < root_num:
```
